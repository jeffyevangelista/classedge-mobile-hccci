# Teacher classroom landing — collapsible hero + tab-aware FAB

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the teacher's flat top-tabs classroom screen with a parallax photo hero that collapses on scroll, sticky tabs underneath, and a Create FAB that only appears on the In-class tab.

**Architecture:** A new `ClassroomScreen` component owns the hero, the collapsible `Tabs.Container` (from `react-native-collapsible-tab-view`), and the tab-aware FAB. The existing list components (`LessonList`, `CourseworkList`, `ClassroomActivitiyList`) get a small `ListComponent` prop so the screen can pass the library's collapsible-aware list variants. The classroom stack layout drops its current header `+` button — the FAB owns the create flow, and the `CreateActionSheet` moves into the screen.

**Tech Stack:** React Native 0.81 · Expo Router 6 · Reanimated 4 · `react-native-collapsible-tab-view` (new dep, v8+ supports Reanimated 4) · `@shopify/flash-list` (existing) · HeroUI Native · pnpm.

## Global Constraints

- Spec scope is A: hero + collapsible behavior + tab-aware FAB only. **Do not** redesign lesson rows, assessment rows, or in-class rows.
- Hero text is `subjectName` (title) and `"{subjectType} · Room {roomNumber}"` (subtitle). No counts. No extra metadata.
- FAB cross-fades (180ms via Reanimated `FadeIn` / `FadeOut`) — no slide-in animation.
- Library is `react-native-collapsible-tab-view`. Do not hand-roll the collapse choreography.
- File-based tab routes go away. The `(tabs)` group must be deleted; routing now goes through a single `index.tsx` at `app/(main)/classroom/[classroomId]/`.
- Auto-staging / auto-committing is disabled per memory. Each task ends with `git status` + a suggested commit message; **the human runs `git add` / `git commit` themselves**.
- Run `pnpm typecheck` (or `pnpm lint`) after each task; both must pass before moving to the next.
- Manual verification is the test mechanism — no automated tests are added.

To swap the inner scrollable from `ClassroomScreen` without forking the list components, we expose the scrollable type as a prop. Default behavior (using the component standalone, like `oversight`/`TeachingCourseList` already does) is unchanged.

- [ ] **Step 1: Create `ClassroomScreen.tsx`**

Create `screens/main/classroom/ClassroomScreen.tsx`:

```tsx
import { Platform, Pressable, StyleSheet, useWindowDimensions, View } from "react-native";
import { useLocalSearchParams, useRouter } from "expo-router";
import { Tabs } from "react-native-collapsible-tab-view";
import { Skeleton, useThemeColor } from "heroui-native";
import { AppText } from "@/components/AppText";
import BackButton from "@/components/BackButton";
import { Icon } from "@/components/Icon";
import Image from "@/components/Image";
import { AttachmentImage } from "@/features/attachments/components/AttachmentImage";
import { useClassroom } from "@/features/classroom/classroom.hooks";
import LessonList from "@/features/classroom/components/LessonList";
import CourseworkList from "@/features/classroom/components/CourseworkList";
import ClassroomActivitiyList from "@/features/classroom/components/ClassroomActivitiyList";

const HERO_RATIO = 0.28; // matches student CourseScreen.tsx:38

const ClassroomScreen = () => {
  const { classroomId } = useLocalSearchParams<{ classroomId: string }>();
  const router = useRouter();
  const { height: screenHeight } = useWindowDimensions();
  const heroHeight = Math.round(screenHeight * HERO_RATIO);

  const surfaceColor = useThemeColor("surface");
  const foregroundColor = useThemeColor("foreground");
  const accentColor = useThemeColor("accent");
  const mutedColor = useThemeColor("muted");

  const { data, isLoading } = useClassroom(classroomId);
  const course = data?.[0];
  const subjectName = course?.subjectName ?? "";
  const subjectType = course?.subjectType ?? "";
  const roomNumber = course?.roomNumber ?? "";
  const subjectPhoto = course?.subjectPhoto ?? undefined;

  const subtitleParts = [subjectType, roomNumber ? `Room ${roomNumber}` : ""].filter(Boolean);
  const subtitle = subtitleParts.join(" · ");

  const renderHeader = () => (
    <View style={{ height: heroHeight }}>
      {isLoading ? (
        <Skeleton style={StyleSheet.absoluteFill} />
      ) : (
        <AttachmentImage
          path={subjectPhoto}
          fallback={
            <Image
              source={require("@/assets/placeholder/bg-placeholder.png")}
              style={StyleSheet.absoluteFill}
              contentFit="cover"
            />
          }
          style={StyleSheet.absoluteFill}
          contentFit="cover"
          cachePolicy="disk"
        />
      )}
      <View style={[StyleSheet.absoluteFill, styles.scrim]} pointerEvents="none" />
      <View style={styles.heroNav}>
        <View style={styles.glassButton}>
          <BackButton tintColor={foregroundColor} />
        </View>
        <Pressable
          onPress={() => router.push(`/(main)/classroom/${classroomId}/course-details`)}
          accessibilityRole="button"
          accessibilityLabel="Open course details"
          style={styles.glassButton}
        >
          <Icon
            name="InfoIcon"
            size={22}
            color={foregroundColor}
            style={{ marginLeft: Platform.OS === "ios" ? -2 : 0 }}
          />
        </Pressable>
      </View>
      <View style={styles.heroText}>
        <AppText weight="semibold" className="text-2xl text-white" numberOfLines={2}>
          {subjectName}
        </AppText>
        {subtitle ? (
          <AppText className="text-xs text-white/90 mt-1" numberOfLines={1}>
            {subtitle}
          </AppText>
        ) : null}
      </View>
    </View>
  );

  const renderTabBar = (props: React.ComponentProps<typeof Tabs.MaterialTabBar>) => (
    <Tabs.MaterialTabBar
      {...props}
      indicatorStyle={{ backgroundColor: accentColor }}
      activeColor={accentColor}
      inactiveColor={mutedColor}
      labelStyle={{ fontFamily: "Poppins-Medium", textTransform: "none" }}
      style={{
        backgroundColor: surfaceColor,
        elevation: 0,
        shadowOpacity: 0,
      }}
    />
  );

  return (
    <Tabs.Container
      renderHeader={renderHeader}
      headerHeight={heroHeight}
      renderTabBar={renderTabBar}
    >
      <Tabs.Tab name="materials" label="Materials">
        <LessonList ListComponent={Tabs.FlatList as never} />
      </Tabs.Tab>
      <Tabs.Tab name="assessments" label="Assessments">
        <CourseworkList ListComponent={Tabs.FlatList as never} />
      </Tabs.Tab>
      <Tabs.Tab name="in-class" label="In-class">
        <ClassroomActivitiyList ListComponent={Tabs.FlashList as never} />
      </Tabs.Tab>
    </Tabs.Container>
  );
};

const styles = StyleSheet.create({
  scrim: {
    backgroundColor: "transparent",
    // linear-gradient via tint layers — keep simple here; tweak in polish if needed
  },
  heroNav: {
    position: "absolute",
    top: 8,
    left: 0,
    right: 0,
    paddingHorizontal: 12,
    flexDirection: "row",
    justifyContent: "space-between",
    alignItems: "center",
  },
  glassButton: {
    width: 40,
    height: 40,
    borderRadius: 9999,
    backgroundColor: "rgba(255,255,255,0.85)",
    alignItems: "center",
    justifyContent: "center",
  },
  heroText: {
    position: "absolute",
    left: 18,
    right: 18,
    bottom: 14,
  },
});

export default ClassroomScreen;
```

> **Notes for the implementer:**
> - The `as never` cast on the `ListComponent` prop is intentional: `Tabs.FlatList`'s exact type (`React.ComponentType<TabsAwareFlatListProps<T>>`) is structurally compatible with our generic `ComponentType<FlatListProps<T>>` but TypeScript's inference can't bridge them. `as never` is the smallest cast that satisfies both ends.
> - The hero uses a translucent black layer for legibility; if the gradient looks weak, drop in `expo-linear-gradient` (already in deps) inside `renderHeader` between the image and the text. Don't over-engineer in this pass — get it functional first.
> - The Reanimated 4 worklet runtime needs `react-native-collapsible-tab-view` to be a peer-compatible version. If the library throws at runtime, downgrade to a known-compatible version (check the library's changelog) and update `package.json`.
