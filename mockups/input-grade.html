Long-name handling — 2-line wrap

Single-line truncation breaks down when several students share a long prefix (siblings, compound last names) — they all truncate to the same characters and become indistinguishable. Using numberOfLines={2} + leading-tight lets the name wrap to a second line so the differentiating part of the name (the first name) is always visible. Row height becomes slightly variable but capped — most rows stay one line; only long-name rows expand to two.

Change summary

Card chrome

  • Soft shadow → hairline border (matches the rest of the app).
  • `bg-surface border border-border rounded-2xl`.

Row layout

  • Two stacked rows → one horizontal row.
  • Avatar · name · score input · max · camera, all left-to-right.
  • ~76pt vs ~140pt → ~7 visible rows per viewport (was ~5).

"Saved" state

  • Drop the inline "Saved" badge on every row.
  • Dim saved rows to opacity 0.55.
  • Filled camera glyph (instead of outline) indicates an attached file.
  • Dirty rows stay full opacity + add an accent left strip.

Progress

  • Drop redundant "100%" — count + bar already say it.
  • Bar turns green at 100%.
  • Optional filter chips on the same row (`All · N` / `Ungraded · N`).

Apply to all

  • Moved from a small text link to a tertiary button in the header.
  • Always reachable regardless of scroll position.

Score input

  • h-10 w-16 → h-11 w-20 — bigger touch target.
  • Font bumped to base/bold for fast scanning while grading.

Search threshold

  • Show search bar when roster has 15+ students (was 10+).
  • Avoids overeager search on small classes.
