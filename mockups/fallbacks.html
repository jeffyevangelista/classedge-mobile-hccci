API

<Fallback
  variant="empty" | "error" | "offline"  // default: "empty"
  density="screen" | "inline"            // default: "screen"
  icon?=IconName                          // optional override
  title?=string                           // sensible default per variant
  description?=string
  action?={{ label: string, onPress: () => void }}
/>

empty

Accent icon chip. For "no rows" surfaces.

error

Danger icon chip. For watch / query failures. Pair with action.

offline

Muted icon chip. For pre-sync surfaces. No action by design.

Migration plan

  1. Add components/Fallback.tsx.
  2. Rewrite the four legacy components as thin shims delegating to <Fallback>. Every existing call site picks up the new look automatically — no consumer touched.
  3. Optional follow-up PR: walk the ~40 call sites to use <Fallback> directly and delete the shims.
