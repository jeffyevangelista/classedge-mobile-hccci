What changes (phone only):
  • Outer screen padding: px-4 → px-3. Gives the body 8pt more horizontal width — wraps fewer lines.
  • Hero internal padding: p-5 → p-4. Saves 8pt vertical without making the card feel cramped (the gradient + dots still breathe).
  • Stat tile padding: p-3 → p-2.5. Numbers stay legible; tile gets shorter.
  • Hero top margin: pt-4 → pt-2 on phone. The back row already provides visual separation.
  • Tabs top margin: mt-4 → mt-3. Minor but adds up.
  • Tab content padding: p-4 → p-3. More content per scroll page.
  • Tab trigger vertical padding: py-2 → py-1.5. Smaller hit target but still ≥44pt for tap.
Net result:

Tab content area gains ~44pt on iPhone SE. That's roughly 2 extra lines of body text visible without scrolling, or one more attempt card visible in the Results tab. Tablet keeps current padding — only the phone branch tightens.
